# 테스트 커버리지 개선 계획

## 현재 상황
- 테스트 커버리지: 미측정 (대부분 0%)
- pre-commit hook을 `--no-verify`로 지속적으로 우회
- 타입 오류 85개 존재
- 테스트 작성이 항상 "나중에"로 밀림

## 목표
- 라인 커버리지 ≥ 85%
- pre-commit hook 정상화
- CI 파이프라인 품질 게이트 설정
- 테스트 작성을 개발 프로세스에 통합

## 주요 개선 사항

### 1. 프로세스 개선
- 기능 티켓과 테스트 티켓 분리 (예: FE-DG-impl, FE-DG-test)
- Definition of Done에 테스트 커버리지 포함
- pre-commit hook 우회 금지

### 2. 기술적 개선
- DataGrid 로직을 퓨어 훅으로 분리
- any 타입 완전 제거
- 테스트 유틸리티 구축

### 3. 도구 및 자동화
- Jest 커버리지 임계값 설정
- 로컬 CI 스크립트 구축
- 커버리지 뱃지 자동 생성

## 작업 단계
1. **INIT-00**: 계획 문서 작성 ✅
2. **RULE-01**: DoD 문서 업데이트
3. **STR-01**: 작업 구조 정리
4. **REF-01/02**: 코드 리팩터링
5. **TEST-CFG**: 테스트 인프라 구축
6. **CI-LOCAL**: 로컬 CI 스크립트
7. **VIS-01**: 커버리지 가시화
8. **LINT-01**: ESLint 규칙 강화
9. **GOAL-01**: 초기 목표 달성
10. **RETRO-01**: 회고 및 개선

## 성공 지표
- DataGrid 컴포넌트 라인 커버리지 ≥ 90%
- BookingService 라인 커버리지 ≥ 90%
- 전체 프로젝트 라인 커버리지 ≥ 85%
- pre-commit hook 100% 통과율
